# 本地同步脚本优化

## 下一步建议

当前默认主线是补齐本地同步脚本，让 code-helper 工具项目在完成新功能后能一键刷新入口文档、内置 skill 模板和三类项目级 skills；随后修复 `Agent协作规范` 的入口索引缺失，运行自动化验证并执行完成检查。

## 目标与约束

- 修复“所有任务交给子代理”的规则没有同步到当前 `AGENTS.md` 的问题。
- 提供脚本化入口，避免后续修改模板后忘记刷新本地 `.code-helper`、`.agents`、`.claude` 和 `.github` 相关内容。
- 同步动作不能额外创建未启用的 `CLAUDE.md` 或 `.github/copilot-instructions.md`。
- 同步动作不能隐式安装 Git hook 或 Agent hook。
- 保持当前项目规则、模板和测试覆盖一致。

## 验收标准

- `npm run sync:local` 能刷新当前项目的 code-helper 入口、内置 skills 和三类项目级 skills。
- 当前 `AGENTS.md` 包含主会话只协调、具体任务交给子代理的核心规则。
- `AGENTS.md` 专题索引包含 `Agent协作规范.md`。
- 同步命令不创建 `CLAUDE.md`、`.github/copilot-instructions.md` 或 hook 配置。
- 自动化测试和协作文档检查通过。
