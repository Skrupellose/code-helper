# 菜单优化

## 下一步建议

当前默认主线已完成：先完成 CLI 主菜单分组、命名和说明，再根据用户反馈优化终端布局清晰度，最后补充测试与文档并运行验证命令。后续如继续扩展菜单，应优先复用主菜单分组数据，并保持非交互子命令行为不变。

## 目标与约束

- 将 CLI 主菜单按“项目准备 / 任务推进 / 项目维护 / 工具设置”分组展示。
- 每个主菜单功能项展示自然中文名称和简短说明。
- 分组标题必须明显区别于功能项，功能名和说明之间要有稳定间距或换行缩进。
- raw mode 交互菜单和非 raw mode 数字菜单共用同一份菜单数据。
- 保持 `init`、`plan`、`manual-test`、`finish`、`archive`、`tasks`、`skills`、`hooks`、`check`、`features` 等非交互命令入口可用。
- 不回滚他人改动，不提交、不推送、不发布 npm。

## 验收标准

- CLI 主菜单按“项目准备 / 任务推进 / 项目维护 / 工具设置”分组展示。
- 每个主菜单功能项显示名称和简短说明，说明列或说明缩进行保持容易扫描。
- README 和长期规则文档沉淀菜单分组、命名和描述规范。
- 非交互子命令入口保持可用。
- 自动化验证命令全部通过。
