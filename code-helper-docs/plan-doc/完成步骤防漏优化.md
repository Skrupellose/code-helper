# 完成步骤防漏优化

## 下一步建议

当前默认主线是把完成检查中的“建议询问”升级为可测试、可输出、可被 agent 明确执行的强制确认清单，随后同步规则模板和项目级 skills，最后运行验证并提交。

## 目标与约束

- 避免任务完成后漏问更新长期记忆、归档文档、选择下一任务等既定步骤。
- 将防漏逻辑落到 `finish` 的结构化结果和 CLI 输出中，不只依赖 agent 自觉。
- 更新当前项目规则、内置模板和项目级 skills，保证新项目和当前项目一致。
- 不自动归档新任务、不自动推送、不自动发布。

## 验收标准

- `finish` 输出必须包含“必须确认事项”。
- ready-to-archive 时必须列出归档确认和归档后任务选择确认。
- 检测到长期规则变更时必须列出更新长期记忆确认。
- 未完成、阻塞、缺文档或节点不完整时必须明确不得询问归档或切换新任务。
- 模板和项目级 skills 与当前实现保持一致。
- 自动化验证通过。
