# 本地同步脚本优化实施记录

## 背景

用户指出 code-helper 工具项目完成新功能后，本地 code-helper 相关内容也需要通过脚本实时更新；当前发现“所有任务交给子代理”的协作规则在本地入口文档中丢失。

## 实施总结

已补齐本地同步脚本和规则传播检查。现在修改模板、入口区块或内置 skills 后，可以通过 `npm run sync:local` 一键刷新当前项目入口、内置 skill 模板和三类项目级 skills。

## 实现

- 新增 `code-helper sync-local`，先刷新本地 code-helper 入口、规则模板和 `.code-helper/skills`，再显式注册 Codex、Claude Code、GitHub Copilot 三类项目级 skills。
- 新增 `npm run sync:local`，串联构建、`sync-local` 和协作文档检查。
- 新增 `code-helper-agent-collaboration` 内置 skill，并注册到 `.agents/skills`、`.claude/skills` 和 `.github/skills`。
- 更新 `AGENTS.md` 受控区块，补回“主会话只做管理、具体任务交给子代理”的核心规则和 `Agent协作规范.md` 索引。
- 更新 README 和 `code-helper开发与发布规范.md`，要求修改模板、入口区块或 agent 协作规则后运行 `npm run sync:local`。

## 验证

- 已通过：`npm run sync:local`
- 已通过：`npm test`
- 已通过：`npm run check`
- 已通过：`npm pack --dry-run`

## 风险与后续

- `sync-local` 会刷新 code-helper 受控区块和受控 skills，不会创建 `CLAUDE.md`、`.github/copilot-instructions.md` 或安装 hooks。
- 后续修改 `src/templates.ts`、入口区块、内置 skills 或 agent 协作规则后，应先运行 `npm run sync:local`，再进入完成检查和提交。
