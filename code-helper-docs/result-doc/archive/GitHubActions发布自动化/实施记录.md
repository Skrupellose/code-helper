# GitHubActions发布自动化实施记录

## 背景

用户确认先提交 review 修复，再按建议将 npm 发布与 GitHub Actions 绑定。实现策略是先建立 CI，再增加手动 npm 发布 workflow，并用 npm Trusted Publishing / OIDC 方向替代长期 npm token。

## 实施总结

已新增 CI 和手动 npm 发布 workflow，并同步 README 与长期发布规则。当前尚未执行实际发布、推送或创建 tag。

## 实现

- 新增 `.github/workflows/ci.yml`：在 PR 和 `main` push 时运行 `npm ci`、`npm test`、`npm run check`、`npm pack --dry-run`，矩阵覆盖 Node.js 20 和 22。
- 新增 `.github/workflows/npm-publish.yml`：通过 `workflow_dispatch` 手动触发，必须输入 `publish`，使用 `id-token: write` 和 `npm publish --access public --provenance`。
- 发布 workflow 会读取当前包名和版本，并在发布前检查 npm 上是否已存在同版本。
- README 增加 GitHub Actions 与 npm trusted publisher 设置说明。
- `code-helper开发与发布规范.md` 增加 CI、手动发布、OIDC 和 provenance 规则。

## 验证

- `npm test`：通过，57 个测试全部通过。
- `npm run check`：通过，未发现协作文档结构问题。
- `npm pack --dry-run`：通过，`prepack` 已触发构建，dry-run 打包成功。

## 风险与后续

- 需要在 npm package 设置中配置 trusted publisher：仓库、`npm-publish.yml` workflow 和 `npm-publish` environment。
- 第一次发布前需要确认 GitHub Environment 是否需要审批人。
- 发布 workflow 不自动改版本；发布前需要先提交版本号变更。
- 本地无法执行远端 GitHub Actions；推送后需要在 GitHub Actions 页面确认 CI 能正常触发。
