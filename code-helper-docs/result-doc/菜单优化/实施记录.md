# 菜单优化实施记录

## 背景

本次需求要求优化 CLI 主菜单的信息架构和文案：按功能用途分组展示，给每个主菜单功能补充说明，优化用户可见命名，并同步 README、项目规则和测试。后续反馈指出菜单仍然“堆在一起、分不清标题和分组”，因此继续优化终端展示布局。已有 `init`、`plan`、`manual-test`、`finish`、`archive`、`tasks`、`skills`、`hooks`、`check`、`features` 等命令入口必须保持可用。

## 实施总结

已完成 CLI 主菜单分组、功能说明、命名同步、布局清晰度优化、测试覆盖和文档沉淀。未发现未完成项。下一步建议是在后续新增菜单项时继续复用同一份主菜单数据，并同步补充菜单布局稳定性测试。

## 实现

- 在 `src/cli.ts` 中抽出主菜单分组数据，分为“项目准备 / 任务推进 / 项目维护 / 工具设置”，raw mode 菜单和非 raw mode 数字菜单共用同一份数据。
- 在 `src/terminal-ui.ts` 中为单选菜单增加不可选分组标题，方向键会跳过分组标题，只能确认具体功能项。
- 在 `src/cli.ts` 中新增主菜单布局格式化函数：分组标题渲染为 `【分组】`，raw mode 功能项使用编号、固定名称列和说明列，数字兜底菜单使用功能名一行加说明缩进行。
- 在 raw mode 菜单的分组之间增加不可选空行，方向键仍跳过空行和分组标题。
- 将主菜单命名更新为“初始化/刷新项目配置、生成任务计划、生成手工测试文档、检查功能完成情况、查看任务列表、归档已完成任务、检查协作规范、功能管理、管理项目 Skills、管理 Hooks”，并把刷新规则模板、入口索引和记忆维护能力合并进“初始化/刷新项目配置”。
- 在 `src/constants.ts`、`src/init.ts`、`src/skills.ts` 中同步用户可见命名，保持底层 feature key 和命令不变。
- 在 `src/completion.ts` 中修正 `finish --check-only` 当前变更路径解析，避免路径首字符丢失或中文路径被显示为转义片段。
- 更新 `README.md`，补充主菜单布局说明、数字兜底菜单示例、主菜单分组表和每个功能项说明。
- 更新 `code-helper-docs/user-rules/code-helper开发与发布规范.md`，沉淀主菜单分组、命名、描述、双菜单同步和布局清晰度规则。
- 更新 `tests/cli.test.js`，覆盖主菜单分组、命名、说明、raw mode 分组选项和布局格式稳定性；更新 `tests/completion.test.js` 覆盖当前变更路径解析；更新 `tests/skills.test.js` 的命名断言。

## 验证

- `npm test`：通过，53 个测试全部通过。
- `npm run check`：通过，未发现协作文档结构问题。
- `npm pack --dry-run`：通过，dry-run 打包成功，包内文件 50 个。
