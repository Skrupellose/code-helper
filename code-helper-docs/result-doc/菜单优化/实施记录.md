# 菜单优化实施记录

## 背景

本次需求要求将一级菜单“项目能力应用”更名为“功能管理”，并让功能管理中的 Skills 和 Agent hooks 应用/取消动作支持按 agent 工具目标执行，同时保持 init 逻辑不倒退。

## 实施总结

已完成 CLI 菜单命名、功能管理目标选择、测试覆盖和文档同步。未发现未完成项。下一步建议是在后续扩展 agent 目标时复用本次解析函数并补充对应测试。

## 实现

- 在 `src/cli.ts` 中将一级菜单和非 raw mode 文本菜单统一改为“功能管理”。
- 为功能管理新增 Skills 目标选择：支持当前项目默认、Codex、Claude Code、GitHub Copilot 和全部。
- 为功能管理新增 Agent hooks 目标选择：支持当前项目默认、Codex、Claude Code 和全部可用 Agent hooks；GitHub Copilot 输入会明确报错。
- 将功能管理应用/取消动作改为按显式目标执行，避免退化为无选择的按当前项目或一键 agent。
- 更新 `README.md` 和 `code-helper-docs/user-rules/code-helper开发与发布规范.md`，沉淀“功能管理”命名与按 agent 目标应用/取消规则。
- 更新 `tests/cli.test.js`，覆盖功能管理目标解析和 GitHub Copilot Agent hook 拒绝逻辑。

## 验证

- `npm test`：通过，49 个测试全部通过。
- `npm run check`：通过，未发现协作文档结构问题。
- `npm pack --dry-run`：通过，dry-run 打包成功。
- `rg -n "项目能力应用" README.md src tests code-helper-docs/user-rules .code-helper/skills .agents/skills .claude/skills .github/skills`：无匹配结果。

## 风险与后续

- 本次没有执行真实交互式方向键菜单手工操作，覆盖重点放在目标解析和 CLI 函数层逻辑。
- 单目标取消 Skills 或 Agent hooks 时不会关闭全局功能开关；按当前项目或全部取消时保留同步关闭后续自动应用能力的既有语义。
